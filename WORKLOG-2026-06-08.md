### Google Integration

- Replaced Supabase as the active backend with Google Apps Script.
- Wishes are submitted to Google Sheets.
- Photos are uploaded to Google Drive.
- Photo metadata is stored in Google Sheets.
- Admin moderation updates Sheet status through Apps Script.

### Immediate Public Display

Changed the public display behavior:

- New wishes and photos appear publicly right away.
- Public pages hide only submissions with status `hidden` or `deleted`.
- Admin can still approve, hide, or delete submissions from `/admin/dashboard`.

### Performance and UX Fixes

Fixed slow UX on `/wall` and `/gallery`:

- Changed `/wall` and `/gallery` to render immediately.
- Added client-side feed components:
  - `components/memory-wall-feed.tsx`
  - `components/photo-gallery-feed.tsx`
- Added loading skeletons so the page does not feel blocked.
- Added browser `sessionStorage` cache for recently loaded wishes/photos.
- Added optimistic recent-item cache after successful wish/photo submission.
- Added public API routes:
  - `app/api/wall/route.ts`
  - `app/api/gallery/route.ts`

Upload performance improvements:

- Compresses large images in the browser before upload.
- Converts large images to JPEG around 1600px max side with reasonable quality.
- Reduces payload sent through Apps Script/base64 to Google Drive.
- Bypasses Next image optimizer for Google Drive images to reduce gallery delay.

### Runtime Fix

Fixed this Next.js runtime error:

```text
Cannot find module './331.js'
```

Cause:

- `.next` cache/chunks were stale after build/dev server changes.

Fix:

- Stopped the old dev server.
- Removed `.next`.
- Restarted `npm run dev`.
